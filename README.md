# YouTube Schedule Viewer

YouTube の登録チャンネルの配信予定・ライブ中の動画を一覧表示する Windows デスクトップアプリです。
Youtubeが日時順にもならなくて一覧性がカスなのにキレて作りました。

![スクリーンショット](https://github.com/harness17/youtube-schedule/releases/download/v1.9.0/screenshot.png)

## 機能

### 予定・ライブ

- 登録チャンネルの配信予定・ライブ中の動画を日付グループで一覧表示
- ライブ中の配信を最上部に表示（赤枠ハイライト）
- 🔔 お知らせ登録 — 気になる配信にフラグを立てると終了後に「見逃し」タブへ自動移動。登録した配信は一覧の先頭に表示 + オレンジ左ボーダーで強調
- ⭐ お気に入り登録 — 登録した配信は一覧の先頭に表示 + オレンジ左ボーダーで強調
- 📌 チャンネル推し設定 — カード上に📌ボタンを常時表示、推し中はゴールドボーダー＋チャンネル名を強調表示して一覧の先頭に
- チャンネル管理モーダル — チャンネル一覧をまとめて検索・推し設定
- チャンネル名クリックで YouTube のチャンネルページを開く
- 配信開始前・ライブ開始時のデスクトップ通知（お知らせ登録した配信のみ。開始前通知は設定で分数変更可）
- タイトル・チャンネル名での検索、チャンネルフィルター（タブ切り替え時に選択を維持）
- **チャンネル自動蓄積** — 登録チャンネル以外の動画でも、一度表示されたチャンネルは次回から自動的に取得対象に追加

### 見逃し・アーカイブ

- **見逃しタブ** — お知らせ登録した予定・配信中の動画と終了済み未視聴を 2 セクションで一覧表示。✓ で視聴済みマーク（お知らせ動画は 90 日間保持）。起動直後からタブバッジに件数表示
- **アーカイブタブ** — 過去配信の全件一覧 + タイトル・チャンネル名・説明文の部分一致検索（日本語対応）・無限スクロール。チャンネルフィルターは登録済み全チャンネルを表示
- **お気に入りタブ** — ⭐ した動画を期限なく保持。予定・配信中・通常（終了済み未視聴）・視聴済みの 3 セクションに自動分類。タブ横のナビボタンで各セクションへジャンプ（その他の過去配信は 30 日で自動削除）。並び替えモードで上下ボタンによる手動並び替えが可能
- 視聴済み動画をアーカイブ・お気に入りタブで薄く表示（既読バッジ）

### 設定（⚙️ ボタン）

- **⚙️ 基本タブ** — ダークモード切り替え・配信開始前通知の分数設定・自動アップデートの確認・ログアウト
- **📌 チャンネルタブ** — チャンネル一覧の検索・推し設定（設定モーダルに統合）
- **📦 データ管理タブ** — 設定（優先チャンネル＋テーマ）のエクスポート・インポート、お気に入り（viewedAt含む）のエクスポート・インポート

### その他

- ダークモード対応（デザインシステムによるテーマ切り替え）
- **簡易モード / フルモード切り替え** — OAuth なしのRSS-only利用と、Google連携による登録チャンネル自動同期をアプリ内で切り替え
- ライブ配信カードの点滅アニメーション
- 自動アップデート（新バージョンが出るとアプリ内バナーで通知）

## インストーラー版を使う（推奨）

### 1. インストーラーをダウンロードする

[Releases](https://github.com/harness17/youtube-schedule/releases) から最新の `youtube-schedule-X.X.X-setup.exe` をダウンロードして実行します。

> Node.js は不要です。

> #### ⚠️ Windows セキュリティの警告が出た場合
>
> インストール時に「**Windows によって PC が保護されました**」と表示されることがあります。
> これはコード署名証明書のないアプリに表示される一般的な警告で、ウイルスではありません。
>
> **回避手順：**
>
> 1. 「**詳細情報**」をクリック
> 2. 「**実行**」ボタンをクリック
>
> ※ コード署名（SignPath.io）の導入作業中です。将来のバージョンでは警告が出なくなる予定です。

### 2. まず簡易モードで使う

初回起動時は **OAuth 認証情報なしで起動できます**。Google Cloud Console の設定は不要です。

1. アプリを起動する（ヘッダーに「簡易モード」バッジが表示されます）
2. 右上の **⚙️ 設定 → 📌 チャンネル** タブを開く
3. **「手動追加」** セクションで以下のいずれかを入力:
   - チャンネルページの URL（例: `https://www.youtube.com/@example`）
   - `@ハンドル名`（例: `@example`）
4. 必要なら表示名を入力（任意）→ **追加** を押す
5. ヘッダーの **↺ 更新** を押すと、新着動画が「新着動画」タブに表示されます

簡易モードは RSS フィードだけを使うため、登録チャンネルの自動同期は行われません。
**配信予定時刻・ライブ配信検出・見逃し追跡** を使いたい場合はフルモードに切り替えてください。

> 簡易モードで取得できるのは「公開済みの動画と配信ページ」のみです。
> RSS にはスケジュール情報が含まれないため、未開始の配信予定は表示されません。

### 3. フルモードで使う（任意）

登録チャンネルの自動同期、配信予定時刻の正確な表示、ライブ配信開始検出を使いたい場合は、
自分専用の **OAuth クライアント** を作成します。

> **所要時間の目安**: 初回 10〜15 分
> **費用**: 無料（YouTube Data API v3 は1日 10,000 ユニットの無料枠で十分動作します）
> **必要なアカウント**: Google アカウント（個人用で OK）

OAuth クライアントは「アプリと Google アカウントの間で安全にやり取りするための鍵」です。
ユーザー自身がオーナーとして発行するため、認証情報がこのアプリの開発者を経由することはありません。

#### 3-1. Google Cloud Console でプロジェクトを作成

1. [Google Cloud Console](https://console.cloud.google.com/) にアクセスし、Google アカウントでログイン
2. 初回はサービス利用規約への同意画面が出るので、チェックを入れて **同意**
3. 画面上部の **プロジェクト選択** ドロップダウン（「Google Cloud」ロゴの右隣）をクリック
4. ダイアログ右上の **「新しいプロジェクト」** をクリック
5. プロジェクト名を入力（例: `youtube-schedule`）→ **作成**
6. 作成完了の通知を待ち、再度プロジェクト選択ドロップダウンから作成したプロジェクトを選ぶ

> プロジェクト名は何でも構いません。アプリ名と一致している必要はありません。

#### 3-2. YouTube Data API v3 を有効化

1. 左の三本線メニュー → **「API とサービス」 → 「ライブラリ」**
2. 検索ボックスに `YouTube Data API v3` を入力 → 表示された結果をクリック
3. **「有効にする」** ボタンをクリック（緑のボタンに変わります）

#### 3-3. OAuth 同意画面を設定

1. 左メニュー **「API とサービス」 → 「OAuth 同意画面」**
2. ユーザーの種類: **「外部」** を選択 → **作成**
3. **アプリ情報** ステップ:
   - アプリ名: 任意（例: `YouTube Schedule Personal`）
   - ユーザーサポートメール: 自分のメールアドレス
   - デベロッパーの連絡先: 自分のメールアドレス
   - その他の項目（アプリのロゴ、ドメイン等）はすべて空欄で OK
   - **保存して次へ**
4. **スコープ** ステップ:
   - **「スコープを追加または削除」** をクリック
   - フィルタ欄に `youtube.readonly` と入力
   - `https://www.googleapis.com/auth/youtube.readonly` にチェックを入れる
   - **更新** → **保存して次へ**
5. **テストユーザー** ステップ:
   - **「+ ADD USERS」** をクリックして、**自分の Google アカウントのメールアドレス** を追加
   - **保存して次へ**
6. **概要** ステップで内容を確認 → **「ダッシュボードに戻る」**

> テストユーザーに追加した Google アカウントでしか認証できません。
> 自分以外の人に配布する予定がなければ、このまま Testing 状態で問題ありません。

#### 3-4. OAuth クライアント ID を作成

1. 左メニュー **「API とサービス」 → 「認証情報」**
2. 画面上部の **「+ 認証情報を作成」 → 「OAuth クライアント ID」** をクリック
3. アプリケーションの種類: **「デスクトップ アプリ」**
4. 名前: 任意（例: `YouTube Schedule Desktop`）
5. **作成** をクリック
6. 表示されたダイアログから **「JSON をダウンロード」** をクリック
   - ダウンロードされたファイル（`client_secret_xxx.json` のような名前）が **credentials.json** として使うものです

> ダウンロードした JSON ファイルには `client_secret` が含まれています。
> 他人に共有したり、公開リポジトリにコミットしたりしないでください。

#### 3-5. アプリに読み込む

1. アプリを起動して右上の **⚙️ 設定** を開く
2. **「⚙️ 基本」** タブの **「動作モード」** セクションへ移動
3. **「credentials.json を読み込み」** をクリック → ファイル選択ダイアログでダウンロードした JSON を選ぶ
4. バッジが「**● credentials.json: 読み込み済み**」（緑）に変わることを確認
5. **「Google連携」** ボタンをクリック → 既定のブラウザで Google ログイン画面が開く
6. テストユーザーに追加した Google アカウントでログイン
7. 「**このアプリは Google で確認されていません**」と表示された場合 → 後述の手順で続行
8. 「**YouTube アカウントの管理**」のスコープを承認
9. ブラウザで「認証完了」と表示されたら、アプリに戻る
10. ヘッダーのバッジが **「フルモード」**（緑）に変わり、登録チャンネルの自動同期が開始されます

> **「このアプリは Google で確認されていません」警告の対処**
>
> あなたが作成した OAuth クライアントは Google の検証を受けていないため、認証時に警告画面が出ます。
> これは仕様上の正常な挙動で、自分が作成したアプリを自分で使うぶんには安全です。
>
> **続行手順**:
> 1. 警告画面の **「詳細」** をクリック
> 2. 表示された **「（アプリ名）に移動（安全ではないページ）」** リンクをクリック
> 3. 通常の Google ログインフローへ進む
>
> この警告を根本的に消すには Google による OAuth app verification（数日〜数週間）が必要です。
> 個人利用が目的の場合は Testing のまま続行で問題ありません。
> 簡易モード（RSS のみ）はそもそも OAuth を使わないので、この警告は表示されません。

#### よくあるトラブル

| 症状 | 原因と対処 |
|------|---------|
| 「credentials.json: 読み込みエラー」と表示される | JSON ファイルが壊れているか、`installed` / `web` キーが含まれていない。手順 3-4 をやり直してダウンロードし直す |
| ログイン時に「アクセスがブロックされました」 | OAuth 同意画面のテストユーザーに自分のメールアドレスが追加されていない。手順 3-3 の 5 を確認 |
| 認証後にチャンネルが表示されない | 該当 Google アカウントが YouTube チャンネルに登録していないか、登録チャンネル数が 0。`⚙️ 設定 → 📌 チャンネル` で確認 |
| クォータ超過エラーバナーが出る | 1 日 10,000 ユニットの上限到達。リセット時刻（PST 0:00 = JST 17:00 ごろ）まで待つ |

## ソースからセットアップ（開発者向け）

### 1. リポジトリをクローン

```bash
git clone https://github.com/harness17/youtube-schedule.git
cd youtube-schedule
npm install
```

### 2. 起動する

```bash
npm run dev
```

開発環境でも簡易モードはそのまま使えます。フルモードを試す場合は、上記「フルモードで使う」の手順で
`credentials.json` を読み込んでください。

> `credentials.json` は `.gitignore` に含まれているため、誤ってコミットされることはありません。

## 注意事項

- YouTube の閲覧専用スコープ（`youtube.readonly`）のみを使用します
- API クォータを節約するため、RSS フィードを優先して使用します
- 簡易モードでは YouTube Data API を使わず、RSS フィードだけで動作します
- `credentials.json` と `token.json` は絶対にリポジトリにコミットしないでください
