# cross-review

反対側エージェントの変更をレビューするためのスキル。

## 観点

- 完成条件を満たしているか。
- 触ってはいけない範囲を変更していないか。
- 認可、所有者チェック、secret、個人情報、quota、公開判断に穴がないか。
- エラー処理、境界値、重複操作、回帰テストに不足がないか。
- verify 結果が handoff と一致しているか。

## 手順

1. `git status --short --branch` と差分を確認する。
2. `CLAUDE_CODE_HANDOFF.md` の該当セクションを読む。
3. 重大指摘を先に列挙する。
4. 問題がなければ、残る検証不足とリスクを短く書く。
5. レビュー結果を `CLAUDE_CODE_HANDOFF.md` に追記する。

## 判定

- 重大指摘あり: merge / publish ブロッカーとして扱う。
- 重大指摘なし: verify 済み範囲と未確認範囲を明示する。
