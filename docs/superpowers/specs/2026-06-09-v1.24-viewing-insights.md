# v1.24 視聴傾向インサイト強化

作成日: 2026-06-09
対象ブランチ: `codex/insight-viewed-rate`

## 目的

既存DBの配信・視聴済み・お気に入り情報を使い、APIクォータやOAuth scopeを増やさず、
チャンネル整理と視聴傾向の振り返りに使える4つのリストを追加する。

## 指標

### よく見る推しチャンネル

- 対象: 現在推し、直近30日に終了したライブ・プレミア
- 視聴済み: `viewed_at IS NOT NULL`
- 表示: 視聴済み件数、全配信数、視聴済み率
- 並び順: 視聴済み件数の多い順、率の高い順、全配信数の多い順
- 視聴済み0件は表示しない

### 未視聴が溜まっているチャンネル

- 対象: 論理削除されていない全チャンネル、直近30日に終了したライブ・プレミア
- 未視聴: `viewed_at IS NULL`
- 表示: 未視聴件数、通知登録件数、最古の未視聴配信日
- 並び順: 未視聴件数の多い順、最古日時の古い順

### 配信頻度 x 視聴済み率

- 対象: 「よく見る推し」と同じ母集団
- 高頻度: 直近30日で4件以上
- 高視聴: 視聴済み率50%以上
- 分類:
  - よく配信・よく見る
  - よく配信・追えていない
  - 配信少なめ・よく見る
  - 配信少なめ・追えていない

### チャンネル別お気に入り傾向

- 対象: DBに保持されている `is_favorite=1` の全動画
- 表示: お気に入り保存数、うち視聴済み件数
- 並び順: 保存数の多い順、視聴済み件数の多い順
- 非お気に入り動画は30日後に削除されるため、お気に入り「率」は算出しない

## UI

- 上位サブナビは既存4項目を維持し、「視聴済み率」を「視聴傾向」へ変更する。
- 視聴傾向内に4つの小タブを置き、一覧を排他表示する。
- チャンネル行はYouTubeチャンネルページを開ける。
- 件数0は各小タブ固有のempty stateを表示する。

## データ契約

既存 `stats:channelActivity` を拡張する。新規IPC channelは追加しない。

- `viewedRates`: 推しチャンネルの30日集計
- `unviewedBacklog`: チャンネル別未視聴集計
- `favoriteChannels`: チャンネル別お気に入り集計

頻度 x 視聴済み率はrendererで `viewedRates` から分類する。

## Non-goal

- DB migration
- YouTube API追加取得
- ジャンル自動分類
- 30日を超える月次・年次推移
